# MyCAE 三人并行开发接口契约

本文档用于复核当前代码和分工计划是否支持三个人同时开发。结论是：现在可以并行，但必须遵守接口冻结和文件归属规则。

## 1. 当前代码是否允许三人并行

可以。

这次已经补齐三条主线共同依赖的基础接口：

```text
MeshBoundary
ResultObject
ResultRepository
SolverCaseWriterResult
SolverCaseWriter
OpenFoamCaseWriter
```

并且已经接入：

```text
ProjectContext
├─ GeometryRepository
├─ MeshRepository
├─ SolverRepository
├─ ResultRepository
└─ SelectionState
```

所以三个人不需要一开始同时修改 `ProjectContext`、`ProjectModel`、`MeshRepository`、`CMakeLists.txt` 来抢公共接口。

## 2. 三条并行主线

### 成员 A：求解器 case 导出与运行

主要目录：

```text
src/solver/export/
src/workflow/SolverCaseWorkflowController.*
src/commands/SolverCaseCommands.*
```

当前可直接接手的接口：

```cpp
class SolverCaseWriter
{
public:
    SolverCaseWriterResult writeOpenFoamCase(
        const ProjectModel &projectModel,
        const QString &caseName
    ) const;
};
```

```cpp
class OpenFoamCaseWriter
{
public:
    SolverCaseWriterResult write(
        const ProjectModel &projectModel,
        const SimulationCase &simulationCase,
        const QString &caseName

```

成员 A 要做的是把当前 placeholder 实现替换为真实 OpenFOAM case 导出逻辑。

成员 A 不需要改：

```text
MeshRepository
ResultRepository
ProjectContext
ProjectModel
```

除非接口确实不够用。

### 成员 B：FaceGroup 到 MeshBoundary / Gmsh physical group

## 3. 必须冻结的公共接口

下面这些接口在第一轮并行开发期间不要随便改字段名。

### 3.1 MeshBoundary 字段冻结

```cpp
QString id;
QString name;
QString meshName;
QString sourceGeometryName;
QString sourceFaceGroupId;
QString physicalGroupName;
int physicalGroupTag;
int faceCount;
```

允许新增字段，但不要删除或重命名已有字段。

### 3.2 ResultObject 字段冻结

```cpp
QString id;
QString name;
QString solverName;
QString casePath;
QString logFile;
QString createdAt;
bool success;
QString summary;
```

允许新增字段，例如 `residualSummary`、`lastTimeStep`，但不要破坏已有字段。

### 3.3 SolverCaseWriterResult 字段冻结

```cpp
bool success;
QString caseRootPath;
QStringList writtenFiles;
QStringList warnings;
QStringList errors;
QStringList logMessages;
```

所有导出/运行相关 workflow 都应该返回或消费这个结构。

## 5. 高冲突文件处理规则

这些文件容易冲突，不建议三个人同时改：

```text
CMakeLists.txt
src/ui/MainWindow.cpp
src/ui/MainWindow.h
src/project/ProjectContext.h
src/project/ProjectContext.cpp
src/project/ProjectModel.h
src/project/ProjectModel.cpp
```

当前已经把第一轮需要的公共接口加进这些文件了。

后续规则：

```text
1. 成员 A/B 尽量不改 MainWindow
2. 成员 A/B 如果新增 command，只提交 command/workflow，不接 UI
3. 成员 C 统一负责 MainWindow 菜单/工具栏接线
4. CMakeLists.txt 每天集中合一次，避免三个人同时改同一区域
5. 如果必须改 ProjectContext/ProjectModel，先在群里说明字段和理由
```

## 6. 并行开发顺序

### 第 0 步：当前已完成

```text
MeshBoundary 接口已存在
ResultObject / ResultRepository 已存在
SolverCaseWriterResult 已存在
SolverCaseWriter / OpenFoamCaseWriter 骨架已存在
Debug 构建通过
```

### 第 1 步：三人同时开工

成员 A：

```text
把 OpenFoamCaseWriter placeholder 改成真实导出：
- 创建 caseRootPath
- 写 0/U
- 写 0/p
- 写 constant/transportProperties
- 写 constant/turbulenceProperties
- 写 system/controlDict
- 写 system/fvSchemes
- 写 system/fvSolution
```

成员 B：

```text
完善 GmshCaseWriter：
- 根据 FaceGroup 生成 physical group 计划
- 明确 OCC face index 到 Gmsh surface tag 的映射策略
- 失败时返回明确错误
- 成功时生成 MeshBoundary
```

成员 C：

```text
完善 Result UI：
- ProjectTreePanel 增加 Results 节点
- PropertyPanel 增加 ResultPropertyView
- 选中 Result 显示 casePath/logFile/success/summary
```

### 第 2 步：第一次集成

集成顺序建议：

```text
1. 先合成员 B 的 MeshBoundary 生成
2. 再合成员 A 的 OpenFoamCaseWriter 使用 MeshBoundary
3. 最后合成员 C 的 UI command 和 Result 展示
```

原因：

```text
OpenFoamCaseWriter 依赖 MeshBoundary
UI 依赖 A/B 的 workflow 入口
```

## 7. 每个人的完成定义

### 成员 A 完成定义

```text
1. OpenFoamCaseWriter 能生成 case 文件夹
2. 缺少 MeshBoundary 时明确失败
3. 缺少 Material/BoundaryCondition 时明确失败
4. writtenFiles 记录所有生成文件
5. Debug 构建通过
```

### 成员 B 完成定义

```text
1. MeshRepository 里能看到 MeshBoundary
2. MeshBoundary 能从 FaceGroup 或 MSH physical group 得到
3. Gmsh 映射失败时不静默通过
4. MeshWorkflowController 生成网格后更新 MeshBoundary
5. Debug 构建通过
```

### 成员 C 完成定义

```text
1. 项目树能显示 Result
2. 属性面板能显示 Result
3. 能从 UI 触发 Export Solver Case
4. 能从 UI 打开 Result 文件夹或 ParaView
5. Debug 构建通过
```

## 8. 当前仍不能完全并行的部分

下面这些任务暂时不要分给多人同时做：

```text
MainWindow 菜单/工具栏重排
ProjectTreePanel 大改
ProjectContext 字段变更
SimulationCase JSON 格式大改
BoundaryCondition 数据结构大改
```

这些属于公共结构，应该由一个人先做接口 PR，其他人再跟。

## 9. 结论

当前代码和计划已经可以支持三人同时开发。

关键原因是：

```text
1. ProjectModel 已经拆成 ProjectContext + Repository
2. MeshBoundary 公共接口已提前存在
3. ResultRepository 公共接口已提前存在
4. SolverCaseWriter 公共接口已提前存在
5. 三条主线的主要文件目录已经分开
```

真正要注意的是：

```text
不要三个人同时改 MainWindow / ProjectContext / CMakeLists.txt。
```

第一轮并行开发应该按下面方式推进：

```text
成员 A：填 OpenFoamCaseWriter
成员 B：填 GmshCaseWriter 和 MeshBoundary 生成
成员 C：填 Result UI 和命令入口
```
