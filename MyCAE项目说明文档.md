## 1. 项目概述

### 1.1 一句话介绍

MyCAE 是一个基于 Qt/C++ 开发的轻量级 CAE 集成系统，目标是把“几何建模、网格划分、材料和载荷定义、有限元求解、结果可视化”这些仿真步骤集成到一个桌面软件中。

通俗地说，MyCAE 想做的是：

```text
让用户在一个软件里完成从模型到仿真结果的完整流程。
```

用户不需要手工写复杂的求解器输入文件，也不需要来回切换很多命令行工具。软件负责把几何、网格、材料、边界条件和载荷组织起来，然后调用外部开源求解器完成计算，最后把结果显示出来。

### 1.2 项目面向的问题

传统 CAE 软件通常存在几个问题：

1. 功能很强，但学习成本高。
2. 软件内部流程复杂，初学者很难理解每一步数据是怎么传递的。
3. 几何、网格、求解器、后处理之间经常是分散的工具链。
4. 想把 CAE 流程和自己的程序、AI Agent、自动化脚本结合时，扩展难度较高。

MyCAE 的设计重点不是堆很多高级功能，而是把 CAE 的主流程做清楚：

```text
建模 -> 选面 -> 分组 -> 网格 -> 材料 -> 约束 -> 载荷 -> 求解 -> 看结果
```

这样做的好处是：

1. 初学者可以理解 CAE 软件的基本工作方式。
2. 老师或同学能看清楚系统每个模块的作用。
3. 后续可以继续扩展 OpenFOAM、AI Agent 自动建模、多求解器调度等功能。
4. 项目结构适合作为课程项目、毕业设计或轻量级 CAE 原型继续开发。

### 1.3 当前项目定位

当前版本的 MyCAE 主要完成了结构有限元分析的基础闭环：

```text
Qt 桌面界面
-> Open CASCADE 几何建模
-> Gmsh 网格划分
-> 材料 / 边界条件 / 载荷定义
-> CalculiX 结构静力求解
-> VTK 结果云图显示
```

其中：

| 名称 | 在项目中的作用 |
|---|---|
| Qt | 构建桌面软件界面、菜单、对话框、工程树 |
| Open CASCADE | 创建和管理三维几何模型 |
| Gmsh | 把几何模型划分成有限元网格 |
| CalculiX | 执行结构有限元计算 |
| VTK | 显示几何、网格和结果云图 |
| CMake | 管理 C++ 工程构建 |

### 1.4 当前已经形成的完整闭环

当前项目可以演示以下完整流程：

```text
创建工程
-> 创建长方体几何
-> 在几何上选择端面
-> 创建面组
-> 设置面组为物理组
-> 生成四面体网格
-> 创建结构材料
-> 把材料分配给实体单元
-> 创建固定约束
-> 创建载荷作用目标
-> 创建集中力 / 压力 / 重力等载荷
-> 导出 CalculiX 输入文件
-> 调用 CalculiX 求解
-> 读取结果文件
-> 显示位移云图和应力云图
```

这就是一个轻量级 CAE 软件最核心的工作链路。

## 2. CAE 基础概念解释

本章先解释一些最基础的词。理解这些词之后，再看软件功能会容易很多。

### 2.1 什么是 CAE

CAE 的全称是 Computer Aided Engineering，即计算机辅助工程。

它的核心思想是：

```text
不用真的把零件造出来，也可以先在电脑里模拟它受力、变形、流动或传热的情况。
```

例如：

1. 一根梁受力后会弯多少？
2. 一个支架哪里应力最大？
3. 一个零件会不会因为受力过大而失效？
4. 一个管道里面流体流动速度如何分布？

当前 MyCAE 主要面向的是结构静力分析，也就是：

```text
物体受到力以后，产生了多大的位移和应力。
```

### 2.2 什么是几何

几何就是要分析的物体形状。

例如：

| 几何类型 | 例子 |
|---|---|
| 长方体 | 梁、板、块体 |
| 圆柱体 | 轴、管、销 |
| 球体 | 球形零件 |
| STEP 模型 | 从 CAD 软件导出的复杂零件 |

在 MyCAE 里，几何由 Open CASCADE 负责创建和保存。

用户看到的是三维窗口中的实体模型；程序内部保存的是 BREP、STEP 和 JSON 等文件。

### 2.3 什么是网格

计算机不能直接对一个连续的三维实体进行有限元求解，所以需要把几何切分成很多小单元，这些小单元就叫网格。

例如一个长方体可以被切成很多四面体：

```text
完整长方体
-> 很多小四面体单元
-> 每个单元上计算位移和应力
-> 汇总成整体结果
```

网格越细，通常结果越精细，但计算速度越慢。

网格越粗，计算速度快，但结果可能不够准确。

在 MyCAE 里，网格主要由 Gmsh 生成。

### 2.4 什么是节点和单元

有限元网格里有两个非常重要的概念：

| 概念 | 解释 |
|---|---|
| 节点 | 网格上的点，可以理解为计算位移的位置 |
| 单元 | 由多个节点组成的小几何块，例如四面体 |

结构分析时，求解器通常会计算每个节点的位移，再根据位移进一步计算单元内的应力。

### 2.5 什么是面组

面组就是把几何模型上的一个或多个面保存成一个有名字的集合。

例如做悬臂梁分析时：

```text
左端面 -> 命名为 Fixed_End
右端面 -> 命名为 Load_End
```

这样后面创建边界条件和载荷时，就不用再临时点选面，而是直接选择已经命名好的面组。

面组的作用非常重要：

1. 它让用户知道哪个面要固定。
2. 它让用户知道哪个面要加载。
3. 它可以被网格模块转成 Gmsh 的 Physical Surface。
4. 它可以被求解器模块转成节点集、面集或边界区域。

### 2.6 什么是物理组

物理组是 Gmsh 里的概念，英文叫 Physical Group。

普通几何面只是几何模型的一部分；物理组则表示这个面在仿真里有特殊意义。

例如：

```text
Fixed_End 这个面组设置为 Physical Surface
```

它的含义是：

```text
这个面不仅是几何上的一个面，而且要在网格文件里保留下来，供后续边界条件使用。
```

如果不设置物理组，网格生成之后可能很难准确知道哪些网格节点属于这个面。

### 2.7 什么是材料

材料用于描述物体本身的力学性质。

结构分析中最常见的材料参数是：

| 参数 | 含义 | 常见单位 |
|---|---|---|
| 弹性模量 E | 材料抵抗变形的能力 | MPa |
| 泊松比 nu | 横向变形和纵向变形的比例 | 无量纲 |
| 密度 rho | 单位体积质量 | tonne/mm^3 或 kg/m^3，取决于单位体系 |

例如钢材常用参数：

```text
弹性模量 E = 210000 MPa
泊松比 nu = 0.30
```

如果使用 mm、N、MPa 这一套单位，E=210000 MPa 是常见写法。

### 2.8 什么是材料分区

材料分区的意思是：

```text
把某种材料分配给某一部分实体单元。
```

例如：

```text
所有实体单元 -> Steel
```

当前项目中最常见、最稳定的用法是把材料分配给全部实体单元，也就是 `EALL`。

`EALL` 可以理解为：

```text
Element All，全部实体单元集合。
```

如果后续要做真正的多材料分区，就需要在几何或网格阶段区分不同体区域，并生成不同的实体单元集合。当前项目已经有部分数据结构基础，但日常演示流程主要使用 `EALL`。

### 2.9 什么是边界条件

边界条件用于告诉求解器：

```text
模型的某些位置受到什么限制。
```

最常见的是固定约束。

例如悬臂梁左端固定：

```text
左端面所有节点的 X/Y/Z 位移都为 0
```

这表示这个面不能移动。

在结构分析里，如果没有边界条件，模型可能会整体刚体运动，求解器无法得到合理结果。

### 2.10 什么是载荷

载荷就是施加在模型上的外部作用。

例如：

| 载荷类型 | 含义 |
|---|---|
| 集中力 Force | 把一个总力施加到目标节点集合上 |
| 表面力 SurfaceForce | 把一个总力等效分配到某个表面节点集合上 |
| 压力 Pressure | 垂直作用在表面上的单位面积力 |
| 重力 Gravity | 对整个模型施加重力场 |

当前项目里，载荷不是直接点一个网格节点施加，而是通过“面组 / 边界目标”来确定作用区域。

### 2.11 什么是求解器

求解器就是实际进行数学计算的程序。

MyCAE 当前结构分析主要调用 CalculiX。

它的工作方式是：

```text
MyCAE 整理工程数据
-> 生成 CalculiX 输入文件 .inp
-> 调用 ccx.exe
-> CalculiX 计算位移和应力
-> 输出 .dat / .frd 等结果文件
-> MyCAE 读取并显示结果
```

### 2.12 什么是后处理

后处理就是查看和分析计算结果。

常见结果包括：

| 结果 | 含义 |
|---|---|
| Ux | X 方向位移 |
| Uy | Y 方向位移 |
| Uz | Z 方向位移 |
| Displacement Magnitude | 总位移大小 |
| Von Mises Stress | 等效应力，常用于判断结构危险区域 |

在 MyCAE 里，后处理主要通过 VTK 显示云图。

## 4. 系统架构与程序设计

### 4.1 总体架构

MyCAE 采用分层设计，主要分为以下几层：

```text
用户界面层
-> 流程控制层
-> 领域数据层
-> 第三方库适配层
-> 文件与外部程序层
```

更具体地说：

```text
Qt UI
-> MainWindow / Dialog / RenderView
-> Controller / Workflow / Command
-> ProjectModel / GeometryObject / MeshData / SimulationCase
-> OCC / Gmsh / CalculiX / VTK Adapter
-> project.json / .step / .msh / .inp / .frd
```

### 4.2 为什么要分层

分层的好处是每一部分只负责自己的事情：

| 层级 | 负责内容 |
|---|---|
| UI 层 | 菜单、按钮、对话框、三维显示 |
| 控制层 | 响应用户操作，组织完整流程 |
| 数据层 | 保存工程、几何、网格、材料、载荷等数据 |
| 适配层 | 对接 OCC、Gmsh、CalculiX、VTK |
| 文件层 | 读写工程文件、网格文件、求解器文件 |

这样设计之后，如果后续要把 CalculiX 换成别的求解器，理论上不需要大改界面；如果要加新的几何创建方式，也不需要改求解器部分。

### 4.3 主要源码目录说明

项目源码大致可以这样理解：

| 目录 | 作用 |
|---|---|
| `src/ui` | 主窗口、菜单、工具栏、对话框、三维渲染界面 |
| `src/geometry` | 几何对象、几何元数据、几何管理 |
| `src/occ` | Open CASCADE 相关封装 |
| `src/mesh` | 网格数据结构、Gmsh 调用、MSH 读取、边界恢复 |
| `src/solver` | 材料、边界条件、载荷、求解器工况 |
| `src/result` | 结果对象、结果字段、后处理数据 |
| `src/workflow` | 高层流程组织，例如从工程数据生成求解工况 |
| `src/commands` | 菜单命令和显示命令 |
| `src/validation` | 数据检查、前置条件检查 |
| `tests` | 自动化测试 |
| `samples` | 示例工程 |

### 4.4 核心数据流

一次完整结构仿真的数据流可以理解为：

```text
用户创建几何
-> GeometryObject 保存几何信息
-> OCC 写出 STEP / BREP
-> Gmsh 根据 STEP 生成 .msh
-> MshReader 读取节点和单元
-> MeshData 保存网格
-> 用户创建材料、边界条件、载荷
-> SimulationCaseBuilder 组装工况
-> CalculiXInputWriter 写出 .inp
-> ccx.exe 执行求解
-> CalculiXResultReader 读取结果
-> VTK 显示云图
```

### 4.5 UI 与数据层关系

用户在界面上点击菜单或填写对话框，本质上是在创建或修改数据对象。

例如：

| 用户操作 | 程序内部结果 |
|---|---|
| 创建长方体 | 新增一个 `GeometryObject` |
| 创建面组 | 新增一个面组数据，记录面 ID |
| 生成网格 | 新增一个 `MeshData` |
| 创建结构材料 | 新增一个材料对象 |
| 创建固定约束 | 新增一个边界条件对象 |
| 创建压力载荷 | 新增一个载荷对象 |
| 运行求解 | 新增一个结果对象 |

所以 MyCAE 不是只做显示，而是把用户操作转换成工程数据。

## 5. 从零完成一次结构仿真的完整流程

本章用一个最适合演示的例子说明项目怎么使用。

推荐算例：

```text
长方体悬臂梁受力分析
```

物理含义：

```text
左端固定，右端向下受力，观察梁的变形和应力分布。
```

### 5.1 推荐单位体系

建议使用：

```text
长度：mm
力：N
应力：MPa，也就是 N/mm^2
弹性模量：MPa
```

这样钢材参数可以直接写：

```text
E = 210000
nu = 0.30
```

### 5.2 新建工程

操作：

```text
文件 -> 新建工程
```

推荐工程名：

```text
cantilever_beam_demo
```

预期结果：

```text
工程树中出现新工程；
工程目录下生成 project.json、geometry、mesh、solver、logs 等目录。
```

### 5.3 创建长方体几何

操作：

```text
几何 -> 创建长方体
```

推荐参数：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 名称 | `Beam_1` | 悬臂梁几何名称 |
| Length | 200 | X 方向长度 |
| Width | 40 | Y 方向宽度 |
| Height | 40 | Z 方向高度 |
| Unit | mm | 单位 |

预期结果：

```text
三维窗口显示一个长方体；
工程树中出现 Beam_1；
geometry 目录下生成几何文件。
```

### 5.4 显示几何边线

操作：

```text
显示 / 仿真相关菜单 -> 几何边线显示
```

作用：

```text
让三维模型的边界轮廓更清楚，便于看出长方体的端面和边线。
```

这不是求解参数，只是显示辅助功能。

### 5.5 创建面组

需要创建两个面组：

| 面组名称 | 选择的面 | 后续用途 |
|---|---|---|
| `Fixed_End` | 左端面 | 固定约束 |
| `Load_End` | 右端面 | 施加载荷 |

操作逻辑：

```text
进入面选择模式
-> 在三维窗口点击端面
-> 从选择结果创建面组
-> 给面组命名
```

注意：

```text
一个面组可以包含一个面，也可以包含多个面。
```

如果你觉得“载荷只能选一个面”不合理，正确做法不是直接给载荷选很多次面，而是：

```text
先把多个面放进同一个面组
-> 再把这个面组作为载荷目标
```

### 5.6 设置面组为物理组

操作：

```text
选中面组 -> 设置面组为物理组
```

作用：

```text
告诉 Gmsh：这个面组在网格文件里要保留下来，后面还要用于边界条件或载荷。
```

如果不设置物理组，网格生成后可能找不到对应的边界面节点。

推荐：

```text
Fixed_End 设置为物理组；
Load_End 设置为物理组。
```

### 5.7 生成网格

操作：

```text
网格 -> 生成网格
```

推荐参数：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 单元类型 | Tetra4 | 一阶四面体，稳定、简单 |
| 最小网格尺寸 | 8 | 不要太小，避免计算太慢 |
| 最大网格尺寸 | 20 | 对 200x40x40 梁比较合适 |
| 自动尺寸 | 可开可关 | 初学演示可先用默认 |

预期结果：

```text
mesh 目录下生成 .msh 文件；
工程树中出现网格对象；
三维窗口显示四面体网格。
```

### 5.8 网格透明显示

当前项目已经支持网格透明显示。

操作：

```text
仿真 -> 网格透明显示
```

作用：

```text
让网格以半透明方式显示，便于看清模型内部单元和背面轮廓。
```

适合演示时使用：

1. 先显示普通网格，让老师看到网格外观。
2. 再打开网格透明显示，说明可以观察内部单元。
3. 再关闭透明显示，说明这是一个显示开关，不影响求解结果。

注意：

```text
网格透明显示只是显示效果，不改变网格数据，也不改变 CalculiX 求解结果。
```

### 5.9 创建结构材料

操作：

```text
工况 -> 结构工况 -> 创建结构材料
```

推荐材料：

```text
Steel
```

推荐参数：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 弹性模量 E | 210000 | MPa，钢材常用值 |
| 泊松比 nu | 0.30 | 钢材常用值 |
| 密度 | 可默认 | 静力位移应力演示时不是重点 |

注意：

```text
要做 CalculiX 结构分析，必须创建结构材料。
```

流体材料主要是为 CFD / OpenFOAM 扩展预留，不能替代结构材料。

### 5.10 创建材料分区

操作：

```text
工况 -> 结构工况 -> 创建材料分区
```

推荐设置：

| 参数 | 推荐值 | 说明 |
|---|---|---|
| 材料 | `Steel` | 前一步创建的结构材料 |
| 几何 | `Beam_1` | 当前长方体 |
| 网格 | `Beam_1_Mesh` 或当前网格 | 当前生成的网格 |
| 单元集 | `EALL` | 全部实体单元 |

材料分区的含义是：

```text
把 Steel 这种材料分配给 EALL 这一组实体单元。
```

如果创建材料分区失败，通常是因为：

1. 没有创建结构材料。
2. 没有创建几何。
3. 没有生成网格。
4. 当前没有可用实体单元集合。

### 5.11 创建固定约束

操作：

```text
工况 -> 结构工况 -> 创建边界条件
```

推荐设置：

| 参数 | 推荐值 |
|---|---|
| 名称 | `BC_Fixed_End` |
| 类型 | FixedSupport 或 Displacement |
| 目标 | `Fixed_End` |
| Ux | 0 |
| Uy | 0 |
| Uz | 0 |

含义：

```text
Fixed_End 这个面上的节点在 X/Y/Z 三个方向都不能移动。
```

这是悬臂梁左端固定的表达方式。

### 5.12 创建载荷目标

当前项目创建载荷时，通常不是直接对几何面施加载荷，而是先创建一个“载荷目标”类型的边界条件。

推荐设置：

| 参数 | 推荐值 |
|---|---|
| 名称 | `BC_Load_End` |
| 类型 | LoadTarget |
| 目标 | `Load_End` |

它的作用是：

```text
把 Load_End 这个面声明为后续载荷可以作用的区域。
```

所以你看到载荷创建时只能选某个目标，是因为软件当前采用的是：

```text
面组 -> 边界目标 -> 载荷
```

这样的数据流程。

### 5.13 创建集中力载荷

操作：

```text
工况 -> 结构工况 -> 创建载荷
```

推荐集中力：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 名称 | `F_Down` | 向下力 |
| 类型 | Force |
| 目标 | `BC_Load_End` |
| X | 0 | X 方向力 |
| Y | 0 | Y 方向力 |
| Z | -1000 | Z 方向向下力 |

这里的 X、Y、Z 表示力的三个方向分量：

```text
X > 0：沿 X 正方向
X < 0：沿 X 负方向
Y > 0：沿 Y 正方向
Y < 0：沿 Y 负方向
Z > 0：沿 Z 正方向
Z < 0：沿 Z 负方向
```

如果长方体的高度方向是 Z，那么向下通常填：

```text
Z = -1000
```

当前项目里，Force 会导出为 CalculiX 的 `*CLOAD`。如果目标区域包含多个节点，程序会把总力等效分配到这些节点上。

### 5.14 创建压力载荷

压力载荷和集中力不一样。

压力的物理含义是：

```text
作用在单位面积上的力，方向通常沿表面法向。
```

所以压力不是 X/Y/Z 三个方向分量，而是一个标量数值。

当前界面里，压力载荷只显示一个输入框，内部暂时复用 `value.x` 保存压力大小。因此你会看到“压力载荷只能有 X”。这里的 X 不是 X 方向，而是：

```text
压力大小数值
```

推荐压力参数：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| 名称 | `P_Load` | 压力载荷 |
| 类型 | Pressure |
| 目标 | `BC_Load_End` |
| 数值 / X | 1.0 | 压力大小，单位可理解为 N/mm^2 |

压力会导出为 CalculiX 的 `*DLOAD`：

```text
surfaceName, P, pressureValue
```

### 5.15 创建重力载荷

重力载荷用于给整个模型施加重力场。

推荐参数：

| 参数 | 推荐值 |
|---|---:|
| X | 0 |
| Y | 0 |
| Z | -1 |
| Magnitude | 9810 或按单位体系设置 |

注意：

```text
重力和密度、单位体系关系更强。
```

如果只是课堂演示结构受力，建议优先演示集中力或压力，重力可以作为扩展功能介绍。

### 5.16 运行 CalculiX 求解

操作：

```text
求解 / 仿真 -> 运行 CalculiX
```

求解前必须具备：

| 条件 | 是否必须 |
|---|---|
| 有几何 | 必须 |
| 有网格 | 必须 |
| 有结构材料 | 必须 |
| 有材料分区 | 必须 |
| 有固定约束 | 必须 |
| 有载荷 | 通常必须 |
| CalculiX 可执行文件可用 | 必须 |

预期结果：

```text
solver 目录下生成 .inp 文件；
CalculiX 运行并输出 .dat / .frd / .sta 等文件；
工程树或结果面板中出现新的结果对象。
```

### 5.17 查看结果

常见查看顺序：

1. 先看 `Displacement Magnitude`，说明整体变形趋势。
2. 再看 `Uz`，说明向下受力后的竖向位移。
3. 再看 `Von Mises Stress`，说明哪里应力集中。
4. 调整变形比例，让变形更明显。
5. 打开网格边线，观察结果和网格关系。
6. 打开未变形轮廓，对比变形前后差异。

注意：

```text
变形比例只是可视化放大，不代表真实变形变大。
```

## 6. 各模块详细说明

### 6.1 工程管理模块

工程管理模块负责把一个仿真项目的所有数据组织起来。

一个工程里通常包含：

```text
project.json       工程基本信息
geometry/          几何文件
mesh/              网格文件
solver/            求解工况和输入输出文件
results/           结果文件
logs/              日志文件
```

工程管理模块的意义是：

1. 防止几何、网格、结果文件散落在不同目录。
2. 让工程可以保存、关闭、重新打开。
3. 让工程树可以显示当前有哪些几何、网格、材料、结果。

### 6.2 几何模块

几何模块负责创建和管理三维模型。

当前支持：

| 功能 | 说明 |
|---|---|
| 创建长方体 | 适合做梁、板、块体演示 |
| 创建圆柱体 | 适合做轴类、管类零件演示 |
| 创建球体 | 适合做基础几何演示 |
| 导入 STEP | 适合导入外部 CAD 模型 |
| 几何显示 | 在 VTK 窗口中显示模型实体 |
| 几何边线显示 | 显示模型轮廓边线 |
| 几何变换 | 平移、旋转、缩放等基础操作 |

“几何显示”就是把当前几何实体显示在三维窗口中。

“几何边线显示”就是在实体表面叠加边线轮廓，方便用户看清楚模型形状和面的位置。

### 6.3 面组模块

面组是当前项目里非常关键的概念。

它连接了：

```text
几何面
-> 网格边界
-> 边界条件
-> 载荷
```

如果没有面组，用户每次创建约束或载荷都要重新点选面，而且点选结果难以保存。

当前面组功能包括：

1. 从三维窗口拾取面。
2. 创建新的面组。
3. 给面组命名。
4. 向已有面组添加面。
5. 从面组移除面。
6. 删除面组。
7. 设置面组为物理组。
8. 作为网格边界恢复依据。
9. 作为边界条件和载荷目标的基础。

### 6.4 网格模块

网格模块负责把几何变成有限元可以计算的数据。

当前网格流程：

```text
几何 STEP 文件
-> Gmsh 生成 .msh
-> MshReader 读取 .msh
-> MeshData 保存节点和单元
-> MeshBoundaryBuilder 恢复边界
-> MeshToVtkConverter 转成 VTK 数据
-> VtkRenderCanvas 显示网格
```

网格参数解释：

| 参数 | 解释 |
|---|---|
| 最小尺寸 | 网格允许的最小单元尺寸 |
| 最大尺寸 | 网格允许的最大单元尺寸 |
| Tetra4 | 一阶四面体，节点少，适合演示 |
| Tetra10 | 二阶四面体，精度更高，但更复杂 |
| 局部网格尺寸 | 对某些面组设置更细或更粗的网格 |

新增的网格透明显示属于显示层功能：

```text
网格数据不变；
只是 VTK actor 的透明度发生变化。
```

### 6.5 材料模块

材料模块负责描述物体的物理性质。

当前结构材料主要支持：

| 参数 | 作用 |
|---|---|
| 弹性模量 | 控制材料刚度 |
| 泊松比 | 控制横向变形关系 |
| 密度 | 和重力、动力学等有关 |

流体材料数据结构也存在，但主要为后续 OpenFOAM / CFD 功能预留。

### 6.6 材料分区模块

材料分区用于回答一个问题：

```text
哪个单元集合使用哪种材料？
```

当前演示中最推荐：

```text
EALL -> Steel
```

含义是：

```text
所有实体单元都使用 Steel 材料。
```

为什么现在通常只能分配给全部单元？

原因是：

1. 要做多材料分区，网格中必须存在多个实体单元集合。
2. 多个实体单元集合通常来自多个体区域或 Gmsh Physical Volume。
3. 当前常规建模流程主要创建的是单一实体和表面物理组。
4. 所以最稳定的结构演示方式是把材料分配给 `EALL`。

这不是材料分区概念错误，而是当前项目的常用工作流还没有把多体材料区域作为主功能完善出来。

### 6.7 边界条件模块

边界条件用于限制模型运动或声明边界用途。

当前结构相关边界条件可以理解为：

| 类型 | 作用 |
|---|---|
| FixedSupport | 固定支撑，位移为 0 |
| Displacement | 指定位移约束 |
| LoadTarget | 声明某个区域可作为载荷目标 |
| Wall | 更偏 CFD / 边界语义预留 |

注意：

```text
FixedSupport 是真正的约束；
LoadTarget 不是约束，它只是告诉载荷可以作用到哪里。
```

### 6.8 载荷模块

载荷模块用于定义外力。

当前载荷类型包括：

| 类型 | 当前含义 | 是否适合演示 |
|---|---|---|
| Force | 总集中力，按目标节点等效分配 | 适合 |
| SurfaceForce | 表面总力，当前也按节点等效处理 | 可演示 |
| Pressure | 压力标量，导出为表面压力 | 适合 |
| Gravity | 重力场 | 可作为扩展 |
| BodyForce | 体力数据结构 | 不建议主线演示 |
| Velocity | CFD 速度边界预留 | 不建议结构主线演示 |
| Temperature | 温度载荷预留 | 不建议主线演示 |

#### 6.8.1 Force 的 X/Y/Z 是什么意思

Force 是向量载荷，所以需要 X、Y、Z 三个方向。

例如：

```text
X = 0
Y = 0
Z = -1000
```

表示：

```text
没有 X 方向力；
没有 Y 方向力；
有一个沿 Z 负方向的 1000 N 力。
```

#### 6.8.2 Pressure 为什么只有一个数值

Pressure 是标量，不是向量。

压力的方向通常由表面法向决定，所以界面只需要输入压力大小。

当前程序内部用 `value.x` 存这个压力大小，所以界面上可能显示为 X 输入框。

正确理解是：

```text
Pressure 的 X 框 = 压力大小，不是 X 方向压力。
```

#### 6.8.3 为什么当前不能直接选点给力

当前项目的载荷目标主要来自面组和边界区域，不是直接来自单个点或单个节点。

也就是说，当前支持的是：

```text
对一个面组 / 边界区域施加载荷
```

暂不支持：

```text
在三维窗口点一个节点，然后直接施加点载荷
```

如果后续要支持点载荷，需要增加：

1. 节点拾取功能。
2. 节点集创建功能。
3. 点载荷目标类型。
4. CalculiX `*CLOAD` 对单节点的精确导出。
5. UI 上的点选反馈和节点编号显示。

### 6.9 CalculiX 求解模块

CalculiX 模块负责把 MyCAE 的工程数据转成 CalculiX 可以识别的输入文件。

主要导出内容包括：

| 内容 | CalculiX 中的含义 |
|---|---|
| 节点 | `*NODE` |
| 单元 | `*ELEMENT` |
| 材料 | `*MATERIAL` |
| 实体截面 | `*SOLID SECTION` |
| 节点集 | `*NSET` |
| 面集 | `*SURFACE` |
| 约束 | `*BOUNDARY` |
| 集中力 | `*CLOAD` |
| 压力 / 重力 | `*DLOAD` |
| 输出请求 | 位移、应力等结果输出 |

这部分是项目从“界面软件”变成“真正能求解”的关键。

### 6.10 后处理模块

后处理模块负责显示结果。

当前主要功能：

1. 读取 CalculiX 结果。
2. 显示位移云图。
3. 显示 Von Mises 应力云图。
4. 切换结果字段。
5. 调整变形比例。
6. 显示网格边线。
7. 显示未变形轮廓。
8. 查询点选位置结果。
9. 显示最大值和最小值。
10. 导出 CSV、报告或截图。

## 7. 实施方案

### 7.1 总体实施思路

项目采用“先打通主流程，再逐步扩展功能”的方式开发。

实施顺序可以概括为：

```text
工程骨架
-> Qt 主界面
-> 几何数据结构
-> OCC 几何创建
-> VTK 几何显示
-> 面选择和面组
-> Gmsh 网格生成
-> 材料 / 约束 / 载荷
-> CalculiX 输入文件导出
-> CalculiX 求解调用
-> VTK 结果后处理
-> 测试和文档
```

### 7.2 关键实施细节

| 功能 | 实施方式 |
|---|---|
| 主界面 | Qt MainWindow、菜单、工程树、属性面板 |
| 三维显示 | VTK 渲染窗口嵌入 Qt |
| 几何建模 | Open CASCADE 创建 TopoDS_Shape |
| 几何保存 | JSON 保存元数据，BREP/STEP 保存几何 |
| 面选择 | VTK/OCC 选择结果映射到面 ID |
| 物理组 | 面组信息传递给 Gmsh |
| 网格生成 | 调用 gmsh.exe 生成 `.msh` |
| 网格读取 | 解析 MSH 节点、单元和物理组 |
| 求解工况 | 统一保存材料、约束、载荷 |
| CalculiX 导出 | 写出 `.inp` 输入文件 |
| 结果读取 | 读取 CalculiX 结果并转换为内部结果对象 |
| 结果显示 | VTK 标量云图、变形显示、色标显示 |

### 7.3 最新功能：网格透明显示

最新项目中加入了网格透明显示开关。

实现思路是：

```text
菜单动作
-> DisplayModeCommand
-> RenderView
-> VtkRenderCanvas
-> 修改网格 actor 透明度
```

用户看到的是：

```text
仿真 -> 网格透明显示
```

开启后，网格以半透明方式显示；关闭后，网格恢复正常不透明显示。

这个功能的价值是：

1. 方便观察内部四面体单元。
2. 方便教学解释网格不是只有外表面。
3. 提升演示效果。
4. 不影响真实网格和求解结果。

## 8. 关键技术与项目亮点

### 8.1 多开源组件集成

项目不是只写一个界面，而是把多个真实 CAE 相关组件集成起来：

```text
Qt + Open CASCADE + Gmsh + CalculiX + VTK
```

这说明项目具备真实工程软件的雏形。

从几何到结果不是孤立功能，而是形成了链路：

几何对象

## 11. 测试与验证

### 11.1 推荐测试算例

推荐使用悬臂梁：

```text
长度 200 mm
宽度 40 mm
高度 40 mm
左端固定
右端向下 1000 N
材料 Steel
E = 210000 MPa
nu = 0.30
```

这个算例简单、稳定、容易解释。

### 11.2 功能验证点

演示时可以逐项验证：

| 验证点 | 成功表现 |
|---|---|
| 工程创建 | 工程树刷新，目录生成 |
| 几何创建 | 三维窗口出现模型 |
| 面组创建 | 工程树出现 Fixed_End / Load_End |
| 物理组设置 | 面组属性显示物理组状态 |
| 网格生成 | 出现四面体网格 |
| 网格透明 | 网格显示从不透明变为半透明 |
| 材料创建 | 工况数据中出现 Steel |
| 材料分区 | Steel 分配给 EALL |
| 固定约束 | Fixed_End 被约束 |
| 载荷创建 | Load_End 有 Force 或 Pressure |
| 求解运行 | 生成 CalculiX 输出文件 |
| 结果显示 | 出现位移和应力云图 |

### 11.3 判断结果是否合理

悬臂梁向下受力后，合理结果通常是：

1. 固定端位移接近 0。
2. 自由端位移最大。
3. 梁整体向下弯曲。
4. 固定端附近应力较大。
5. Von Mises 应力云图在固定端附近更明显。

如果结果完全不变形，可能是：

1. 载荷没有施加成功。
2. 结果字段没有切到位移。
3. 变形比例太小。
4. 求解失败但结果没有更新。

如果模型刚体运动或求解失败，可能是：

1. 没有固定约束。
2. 材料分区缺失。
3. 网格边界没有正确恢复。
4. CalculiX 输入文件不完整。

## 13. 常见问题解释

### 13.1 “几何显示”在哪里

几何显示一般发生在：

```text
创建几何之后；
打开已有几何之后；
从工程树选择几何对象之后。
```

它指的是把几何实体显示到中间的三维视图里。

如果三维窗口里看到长方体、圆柱体、球体或 STEP 模型，这就是几何显示。

### 13.2 “几何边线显示”是干什么的

几何边线显示只是一个辅助观察功能。

作用是：

1. 看清模型轮廓。
2. 看清端面边界。
3. 方便选面。
4. 方便演示几何结构。

它不会影响网格，也不会影响求解。

### 13.3 “设置面组为物理组”是干什么的

它的作用是把你选出来的面组告诉 Gmsh：

```text
这个面很重要，网格生成后还要保留它的边界身份。
```

如果后面要在这个面上固定或加载，建议设置为物理组。

### 13.4 为什么载荷创建时好像只能选一个面

当前载荷通常不是直接选几何面，而是选择一个边界目标。

如果你想让多个面一起受力，推荐做法是：

```text
把多个面加入同一个面组
-> 设置为物理组
-> 创建 LoadTarget
-> 对这个 LoadTarget 创建载荷
```

所以界面上看起来是选一个目标，但这个目标内部可以包含多个面。

### 13.5 为什么压力载荷只有 X

压力是标量，不是 X/Y/Z 向量。

当前界面复用了 X 输入框保存压力数值，所以显示上容易误解。

正确理解：

```text
压力载荷里的 X = 压力大小
```

不是：

```text
沿 X 方向的压力
```

### 13.6 为什么现在不能选点给力

当前项目暂时没有实现点选节点施加载荷。

现在支持的是：

```text
面组 / 边界区域载荷
```

不支持：

```text
单个点 / 单个节点载荷
```

这是后续可以扩展的功能。

### 13.7 为什么无法创建材料分区

通常检查这几项：

1. 是否已经创建结构材料。
2. 是否已经创建几何。
3. 是否已经生成网格。
4. 是否有可用实体单元集合。
5. 是否选择了 `EALL`。

如果没有结构材料，材料分区没有材料可选。

如果没有网格，材料分区不知道要分配给哪些单元。

### 13.8 材料现在是不是只能分配给全部单元

当前常用演示流程里，基本是：

```text
Steel -> EALL
```

也就是分配给全部实体单元。

这适合单材料结构分析。

如果要做多个材料区域，需要后续完善：

1. 多实体几何管理。
2. Physical Volume。
3. 单元集可视化。
4. 每个单元集分配不同材料。
5. 多个 `*SOLID SECTION` 导出。
