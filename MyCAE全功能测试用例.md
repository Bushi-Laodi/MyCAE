# MyCAE 全功能测试用例

## 1. 测试目标

本文档用于覆盖 MyCAE 当前版本已经具备的主要功能，验证系统是否能够完成轻量级 CAE 软件的核心流程：

```text
工程管理
-> 几何建模
-> 面拾取 / 面组
-> 网格划分
-> 材料 / 边界条件 / 载荷
-> CalculiX 求解
-> VTK 后处理显示
-> 导出与自动化验证
```

测试重点包括：

- 功能是否可点击、可执行；
- 执行后工程树、属性面板、日志面板是否正确更新；
- 生成的几何、网格、工况、结果文件是否存在；
- 三维显示是否符合预期；
- 求解器流程是否能够导出、运行、读取结果；
- 自动化验证命令是否能通过。

## 2. 测试环境

推荐测试环境：

```text
操作系统：Windows 10 / Windows 11
编译器：MSVC 64-bit
界面框架：Qt 6.6.x
构建工具：CMake
几何内核：Open CASCADE
网格工具：Gmsh
求解器：CalculiX
可视化库：VTK
```

程序路径示例：

```powershell
.\build-codex-qt663-ninja\MyCAE.exe
```

自动化验证命令：

```powershell
ctest --test-dir .\build-codex-qt663-ninja --output-on-failure
```

## 3. 测试数据准备

建议准备以下测试数据：

```text
1. 空白新工程目录
2. 样例工程：samples/projects/box_pressure_demo/project.json
3. 可导入 STEP 文件，例如 box_1.step
4. 已安装并可运行的 gmsh.exe
5. 已安装并可运行的 ccx.exe
```

推荐测试顺序：

```text
先运行样例工程验证
-> 再创建新工程测试建模
-> 再测试网格和求解
-> 最后测试导出和自动化命令
```

## 4. 测试结果标记

测试记录时可使用以下状态：

```text
PASS：通过
FAIL：失败
BLOCKED：环境或依赖缺失，无法执行
SKIP：当前版本暂不测试
```

## 5. 工程管理功能测试

### TC-PROJ-001 新建工程

前置条件：

- MyCAE 程序正常启动；
- 选择一个不存在 `project.json` 的空目录。

测试步骤：

1. 点击“文件 -> 新建工程”。
2. 选择或输入新工程目录。
3. 确认创建工程。

预期结果：

- 工程创建成功；
- 工程目录下生成 `project.json`；
- 工程目录下生成 `geometry`、`mesh`、`solver`、`logs` 等子目录；
- 工程树刷新并显示当前工程；
- 日志面板显示工程创建成功信息。

### TC-PROJ-002 打开已有工程

前置条件：

- 已存在一个有效 MyCAE 工程；
- 工程目录中存在 `project.json`。

测试步骤：

1. 点击“文件 -> 打开工程”。
2. 选择已有工程的 `project.json`。
3. 确认打开。

预期结果：

- 工程加载成功；
- 工程树显示几何、网格、材料、边界条件、载荷和结果；
- 属性面板可显示选中对象信息；
- 日志面板显示打开工程成功信息。

### TC-PROJ-003 打开样例工程

前置条件：

- 样例工程目录存在：

```text
samples/projects/box_pressure_demo/project.json
```

测试步骤：

1. 点击“文件 -> 打开工程”。
2. 选择 `box_pressure_demo/project.json`。
3. 展开工程树节点。

预期结果：

- 样例工程打开成功；
- 工程树中可看到几何、网格、工况和结果数据；
- 选择不同节点时，属性面板同步更新。

### TC-PROJ-004 最近工程记录

前置条件：

- 至少打开过一个工程。

测试步骤：

1. 点击“文件 -> 最近工程”。
2. 查看最近工程列表。
3. 点击一个最近工程。

预期结果：

- 最近工程列表显示历史工程路径；
- 点击后可重新打开对应工程；
- 若工程路径失效，应在日志或提示中说明打开失败。

### TC-PROJ-005 工程资源查看

前置条件：

- 已打开工程。

测试步骤：

1. 点击“工具 -> 工程资源”。
2. 查看资源列表。

预期结果：

- 弹出工程资源窗口；
- 能看到工程中的几何、网格、求解和结果文件；
- 文件路径、类型或大小信息显示合理。

## 16. 风险与注意事项

### 16.1 外部依赖风险

Gmsh、CalculiX、Qt、VTK、Open CASCADE 均为外部依赖。如果路径不正确或 DLL 缺失，相关功能可能无法运行。

检查顺序：

```text
Qt 平台插件
-> VTK DLL
-> OCC DLL
-> Gmsh 路径
-> CalculiX 路径
-> Debug / Release 是否混用
```

### 16.2 现场演示风险

现场演示时建议优先打开样例工程，不要从零执行复杂求解。

推荐演示策略：

```text
先展示样例工程完整结果
-> 再演示创建几何和显示网格
-> 最后介绍求解器插件和自动化验证
```

### 16.3 当前 CFD 功能边界
