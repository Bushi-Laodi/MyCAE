---

**一、案例级输入**

软件界面最上层可以叫：

```text
案例类型：Stress Concentration
分析类型：Linear Static
建模维度：2D 平面模型
单位体系：mm, N, MPa
求解器：CalculiX
```

案例有两个加载实现方式：

```text
方案 A：Stress
加载方式：Rigid Body + Rot Node Moment
对应文件：par.run.fbl + Stress.inp

方案 B：Stress1
加载方式：Kinematic Coupling + Surface Moment
对应文件：par.run1.fbl + Stress1.inp
```

如果你们第一版要复现，建议先选 **方案 A：Stress**，因为它不需要生成 `*SURFACE`，只要节点集和刚体约束。

---

**二、几何建模输入**

来自 `par.run.fbl` / `par.run1.fbl`：

```text
L = 100 mm
W = 60 mm
R = 15 mm
```

含义：

```text
L：从孔中心线到加载端的长度
W：板/梁总宽度
R：圆孔半径
```

实际建模采用半模型，再关于 Y 方向镜像。

原始点定义：

```text
p1 = (L, 0, 0)       = (100, 0, 0)
p2 = (R, 0, 0)       = (15, 0, 0)
p3 = (0, 0, 0)       = (0, 0, 0)
p4 = (0, R, 0)       = (0, 15, 0)
p5 = (0, W/2, 0)     = (0, 30, 0)
p6 = (L, W/2, 0)     = (100, 30, 0)
```

边线定义：

```text
l1：p1 -> p2
l2：p2 -> p4，以 p3 为圆心的圆弧
l3：p4 -> p5
l4：p5 -> p6
l5：p6 -> p1
```

几何轮廓就是：

```text
右半模型：
x = 0 到 x = L
y = 0 到 y = W/2
左侧有四分之一圆孔边界

然后关于 y=0 镜像，得到上下对称的半板模型。
```

对应 CGX 操作：

```text
surf ! l1 l2 l3 l4 l5
flip all
copy all new mir y
merg n all
```

如果用 OCC/Gmsh 重做，几何输入可以简化成：

```text
长度 L = 100
宽度 W = 60
孔半径 R = 15
厚度 t = 1
建模方式：2D 面模型或厚度为 1 的薄实体
对称方式：只建 x >= 0 的半模型
```

**三、网格划分输入**

来自 `.fbl` 文件。

方案 A `Stress`：

```text
l1 divisions = 20, growth = 0.9
l2 divisions = 30, growth = 0.9
l3 divisions = 20, growth = 1.0
l4 divisions = 30, growth = 1.1
l5 divisions = 8
```

方案 B `Stress1` 相同。

单元类型流程：

```text
elty all tr6u
mesh all
elty all tr6s
```

也就是说它使用的是 **6 节点二次三角形单元**，最后转为 `tr6s` 类型。

网格相关输入可以整理为：

```text
单元类型：TR6 二次三角形
分析假设：平面应力/二维应力输出
孔边圆弧分段数：30
右侧底边分段数：20
左侧竖边分段数：20
上边分段数：30
右端竖边分段数：8
网格增长率：
  l1 = 0.9
  l2 = 0.9
  l3 = 1.0
  l4 = 1.1
```

两个变体的区别：

```text
Stress:
asgn n 3

Stress1:
asgn n 2
```

这属于 CGX 脚本里的网格/属性设置。你们用 Gmsh 重做时，不一定要保留这个命令，但要保留核心意图：

```text
孔边加密
圆弧二次单元
整体二次三角网格
```

**四、集合 / 区域输入**

这个案例必须生成这些区域。

方案 A `Stress`：

```text
sym：
x = 0 附近的节点
选择条件：rec 0 _ 0 0.0001
用途：对称边界，约束 X 位移

rb：
x = L 附近的节点
选择条件：rec L _ 0 0.0001
用途：刚体区域，用于加载端刚体约束

nref：
参考节点 1
坐标：(L, 0, 0)

nrot：
旋转节点 2
坐标：(L, 0, 0)
```

方案 B `Stress1`：

```text
sym：
x = 0 附近的节点
用途：对称边界，约束 X 位移

load：
x = L 附近的加载表面
用途：运动耦合 surface

nref：
参考节点 1
坐标：(L, 0, 0)
```

如果做成软件输入，应该暴露：

```text
对称边位置：x = 0
加载边位置：x = L
选择容差：0.0001
参考点坐标：(L, 0, 0)
加载区域类型：节点集 / 表面集
```

**五、材料输入**

两个 `.inp` 完全一致：

```text
材料名：steel
弹性模量 E = 210000 MPa
泊松比 nu = 0.3
密度 rho = 7.85e-9
```

CalculiX 写法：

```inp
*material, name=steel
*elastic
210000,0.3
*density
7.85e-9
```

密度在线性静力里不是核心必需项，但案例里写了，所以复现时应保留。

**六、截面 / 厚度输入**

两个 `.inp` 都有：

```inp
1
```

这里的 `1` 对应厚度/截面参数。

结合 `Stressconc1.sm`，厚度输入是：

```text
t = 1 mm
```

所以软件里应暴露：

```text
厚度 t = 1 mm
单元/截面集合：Eall

```

**七、载荷输入**

SMath 文件里说明：

```text
The cantilever beam is loaded such that the specified nominal stress is reached.
```

参数：

```text
名义应力 sigma_nom = 1 MPa
截面宽度 W = 60 mm
厚度 t = 1 mm
```

脚本里直接用：

```text
M = W * W / 6
```

代入：

```text
M = 60 * 60 / 6 = 600
```

如果考虑厚度和名义应力，物理含义是：

```text
M = sigma_nom * t * W^2 / 6
```

在当前案例：

```text
sigma_nom = 1 MPa
t = 1 mm
W = 60 mm
M = 600 N*mm
```

方案 A `Stress` 的载荷：

```inp
*cload
2,3,600
```

含义：

```text
节点：2，也就是 nrot
自由度：3
载荷值：600
```

方案 B `Stress1` 的载荷：

```inp
*cload
1,6,600
```

含义：

```text
节点：1，也就是 nref
自由度：6
载荷值：600
```

所以软件输入应有：

```text
加载方式：
- Rigid Body Moment
- Kinematic Coupling Moment

目标名义应力 sigma_nom = 1 MPa
或直接输入弯矩 M = 600 N*mm

加载参考点：
x = L
y = 0
z = 0

加载自由度：
Stress：node 2, dof 3
Stress1：node 1, dof 6
```

**八、边界条件输入**

方案 A `Stress.inp`：

```inp
*rigid body, nset=Nrb, ref node=1, rot node=2

*boundary
Nsym, 1
1,2,3
2,1,2
```

整理成软件输入：

```text
刚体约束：
节点集：Nrb
参考节点：1
旋转节点：2

对称边界：
节点集：Nsym
约束自由度：1
含义：Ux = 0

参考节点约束：
节点 1
约束自由度：2 到 3
含义：Uy = 0, Uz = 0

旋转节点约束：
节点 2
约束自由度：1 到 2
含义：U/R 对应自由度 1,2 约束
```

方案 B `Stress1.inp`：

```inp
*coupling, surface=Sload, ref node=1,constraint name=load
*kinematic
1,3

*boundary
Nsym, 1
```

整理成软件输入：

```text
耦合约束：
表面：Sload
参考节点：1
约束名称：load
类型：kinematic
耦合自由度：1 到 3

对称边界：
节点集：Nsym
约束自由度：1
含义：Ux = 0
```

**十、结果输出输入**

方案 A `Stress`：

```inp
*node file
U
*el file, output=2D
S
*node print, nset=rotnode
U
```

软件上对应：

```text
节点文件输出：U 位移
单元文件输出：S 应力
应力输出形式：2D
文本输出节点集：rotnode
文本输出量：U
```

方案 B `Stress1`：

```inp
*node file
U,RF
*el file, output=3D
S
*section print, surface=Sload,name=load
SOF
```

软件上对应：

```text
节点文件输出：U 位移、RF 反力
单元文件输出：S 应力
应力输出形式：3D
截面输出表面：Sload
截面输出名：load
截面输出量：SOF
```

如果你们现在要做第一个可复现版本，我建议严格按 **`Stress` 方案** 做，不先做 `Stress1`。它的软件输入更少：

```text
几何：L, W, R, t
材料：E, nu, density
网格：5 条边的 divisions/growth，TR6
集合：Nsym, Nrb, nref, nrot
约束：rigid body + symmetry
载荷：M = 600 N*mm
求解：linear static
输出：U, S
```

这就是这个案例从建模、网格到求解器涉及的全部关键输入。
