# MyCAE 当前功能测试流程

本文档用于提交前手动验证 MyCAE 当前已经实现的功能。测试目标不是验证最终 CAE 求解精度，而是确认当前软件主流程可以稳定运行：

```text
工程创建/打开
-> 几何创建与显示
-> OCC BREP/STEP 文件生成
-> Gmsh 网格生成
-> MSH 读取与 VTK 网格显示
-> 材料/边界条件/载荷输入
-> SimulationCase 持久化
-> 外部求解器插件 demo 导出与运行
```

## 1. 测试前准备

### 1.1 环境要求

当前项目按以下环境配置：

```text
Qt: 6.6.3, MSVC 2019 64bit
VTK: 9.4.2
Open CASCADE: 7.9.3
Gmsh: 4.15.2
构建目录: build-codex-qt663-ninja
```

### 1.2 推荐测试工程目录

建议新建一个空目录作为测试工程，例如：

```text
D:/study/Qt_code/cae/test_projects/MyCAE_Test_01
```

不要直接选择源码目录作为工程目录。

### 1.3 构建验证

在项目根目录执行：

```powershell
cmake --build build-codex-qt663-ninja --config Debug
```

预期结果：

```text
构建成功
生成 MyCAE.exe
无 C++ 编译错误
```

## 2. 启动测试

启动：

```text
build-codex-qt663-ninja/MyCAE.exe
```

预期结果：

```text
主窗口正常打开
左侧显示 Project / Model 工程树
右侧显示 Properties 属性面板
底部显示 Log 日志面板
中央 VTK 显示区正常初始化
日志出现 MyCAE started / VTK renderer initialized
```

## 3. 工程功能测试

### 3.1 新建工程

操作：

```text
File -> New Project
选择一个空目录
```

预期结果：

```text
工程树根节点显示工程名
工程目录下生成 project.json
工程目录下生成 geometry/
工程目录下生成 mesh/
工程目录下生成 solver/
工程目录下生成 logs/
工程目录下生成 solver/case.json
日志显示 Project created
日志显示 Simulation case saved: solver/case.json
```

### 3.2 打开工程

操作：

```text
关闭软件
重新启动 MyCAE
File -> Open Project
选择刚才工程目录下的 project.json
```

预期结果：

```text
工程树恢复工程名
已有 geometry / mesh / solver 数据被重新加载
日志显示 Project opened
日志显示 Loaded simulation case data
```

## 5. Gmsh 网格功能测试

### 5.1 Gmsh 环境检查

操作：

```text
Mesh -> Check Gmsh
```

预期结果：

```text
日志显示 Gmsh path
日志显示 Gmsh command
日志显示 Gmsh stdout 或版本号
日志显示 Gmsh environment check succeeded
```

如果失败，优先检查：

```text
CMakeLists.txt 中 MYCAE_GMSH_EXECUTABLE 是否指向正确 gmsh.exe
```

### 5.2 生成网格

操作：

```text
在工程树中选择 Box_1 或 Cylinder_1
Mesh -> Generate Mesh
```

预期结果：

```text
日志显示 Gmsh command
工程目录 mesh/ 下生成对应 .msh 文件
工程目录 mesh/ 下生成对应 mesh json 文件
工程树 Mesh 下出现 xxx_Mesh
solver/case.json 被同步更新
```

当前限制：

```text
MshReader 当前主要读取 Gmsh msh2 ASCII
当前 VTK 转换重点支持 4 节点四面体 elementType == 4
```

### 5.3 读取网格信息

操作：

```text
选择已有几何
Mesh -> Read Mesh Info
```

预期结果：

```text
日志显示 MSH file 路径
日志显示 Node count
日志显示 Tetra count
```

### 5.4 显示网格

操作：

```text
选择已有几何
Mesh -> Show Mesh
或点击工程树 Mesh 下的 xxx_Mesh
```

预期结果：

```text
中央 VTK 区域显示网格
右侧属性面板显示 MeshObject 信息
日志显示 Mesh displayed
```

## 6. 求解前处理数据测试

### 6.1 创建材料

操作：

```text
Solver Setup -> Create Material
输入示例：
  Name: Water
  Domain: Fluid
  Enable Density: true
  Density: 1000 kg/m^3
  Enable Dynamic Viscosity: true
  Dynamic Viscosity: 0.001 Pa*s
点击 OK
```

预期结果：

```text
日志显示 Material created
日志显示 Simulation case saved: solver/case.json
点击工程树 Materials
右侧属性面板显示 Water
solver/case.json 中 materials 数组包含 Water
```

重复输入同名材料时，预期：

```text
日志提示 duplicated material ID
不会新增重复材料
```

### 6.2 创建边界条件

操作：

```text
Solver Setup -> Create Boundary Condition
输入示例：
  Name: Inlet
  Type: Velocity Inlet
  Geometry Name: Box_1
  Face Group: Inlet
  Material ID: water
点击 OK
```

预期结果：

```text
日志显示 Boundary condition created
点击工程树 Boundary Conditions
右侧属性面板显示 Inlet
solver/case.json 中 boundaryConditions 数组包含 Inlet
```

重复输入同名边界条件时，预期：

```text
日志提示 duplicated boundary condition ID
不会新增重复边界条件
```

### 6.3 创建载荷

操作：

```text
Solver Setup -> Create Load
输入示例：
  Name: Inlet Velocity
  Type: Velocity
  Boundary Condition ID: inlet
  Field Name: U
  Value: 0.8
  Unit: m/s
点击 OK
```

预期结果：

```text
日志显示 Load created
点击工程树 Loads
右侧属性面板显示 Inlet Velocity
solver/case.json 中 loads 数组包含 Inlet Velocity
```

重复输入同名载荷时，预期：

```text
日志提示 duplicated load ID
不会新增重复载荷
```

### 6.4 Solver 分类显示

操作：

```text
点击工程树 Solver
```

预期结果：

```text
右侧属性面板显示当前 SimulationCase
显示 Case ID
显示 Source Geometry
显示 Mesh
显示 Solver Type
显示 Turbulence Model
显示 Materials 数量
显示 Boundary Conditions 数量
显示 Loads 数量
```


## 8. 求解器插件 Demo 测试

### 8.1 插件菜单检查

操作：

```text
打开 Simulation 菜单
```

预期结果：

```text
如果 resource/solver/demo_solver/solver_plugin.json 已正确复制
菜单中应出现 Run External Demo Solver
```

如果显示：

```text
No solver plugins found
```

优先检查：

```text
build 输出目录下是否存在 resource/solver/demo_solver/solver_plugin.json
CMake 自定义目标 MyCAEResources 是否执行
```

### 8.2 运行 Demo Solver

操作：

```text
Simulation -> Run External Demo Solver
```

预期结果：

```text
日志显示 Solver plugin
日志显示 Solver case directory
日志显示 Solver case data: x materials, y boundary conditions, z loads
日志显示 Solver input exported
日志显示 Solver result
```

工程目录中预期生成：

```text
solver/demo/case.json
solver/demo/result.json
```

其中：

```text
solver/case.json
  是工程级求解输入持久化文件

solver/demo/case.json
  是 demo 插件运行时导出的输入文件

solver/demo/result.json
  是 demo 插件输出结果
```
